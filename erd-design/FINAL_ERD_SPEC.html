2. 핵심 테이블 상세 설계

📊 interactions (통합 상호작용)

댓글, 오답노트, 좋아요를 한 테이블에서 관리하며 대댓글 구조를 지원합니다.

컬럼명타입속성설명
idintPK식별자
material_idintFK INDEX대상 학업 자료
parent_idintFK부모 상호작용 (대댓글용)
typevarcharINDEXCOMMENT, NOTE, LIKE 구분

📊 schedules (통합 일정)

개인 일정과 그룹 공유 일정을 통합하여 관리합니다.

컬럼명타입속성설명
idintPK식별자
user_idintFK소유자
group_idintFKNULL이면 개인, 존재하면 그룹 일정

3. dbdiagram.io 전용 DBML 코드

// [dbdiagram.io 복사용 최종 코드] Table users { id int [pk, increment] login_id varchar [unique, not null] email varchar [unique, not null] nickname varchar [not null] status varchar [default: 'ACTIVE'] } Table materials { id int [pk, increment] user_id int [ref: > users.id] category varchar title varchar [not null] status varchar [default: 'PUBLISHED'] } Table interactions { id int [pk, increment] user_id int [ref: > users.id] material_id int [ref: > materials.id] parent_id int [ref: > interactions.id] type varchar [not null] indexes { (material_id, type) } } Table groups { id int [pk, increment] unique_code varchar [unique, not null] name varchar [not null] creator_id int [ref: > users.id] } Table members { group_id int [ref: > groups.id] user_id int [ref: > users.id] role varchar [default: 'MEMBER'] indexes { (group_id, user_id) [pk] } } Table chats { id int [pk, increment] group_id int [ref: > groups.id] user_id int [ref: > users.id] message text [not null] indexes { (group_id, sent_at) } } Table schedules { id int [pk, increment] user_id int [ref: > users.id] group_id int [ref: > groups.id] title varchar [not null] start_at datetime [not null] end_at datetime [not null] }
